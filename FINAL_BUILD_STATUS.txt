### 1. SocialPost Ambiguity ‚úÖ

**Error**: `'SocialPost' is ambiguous for type lookup in this context`

**Location**: `ViewModels/SocialViewModel.swift`

**Problem**: The `typealias SocialPostModel = SocialPost` wasn't resolving the ambiguity

**Solution**: Removed the typealias and used `SocialPost` directly

```swift
// Before (with typealias - didn't work)

var posts: [SocialPostModel] = []

// After (direct reference - works!)

```

**Why it works**: Using the type name directly forces the compiler to resolve it correctly without intermediate aliasing that can confuse the macro system.

### 2. onAppear on Conditional View ‚úÖ

**Error**: `Instance member 'onAppear' cannot be used on type 'View'`

**Location**: `Views/Activity/ActivityView.swift:257`

**Problem**: `.onAppear` was being applied to the result of an `if showAICoach { }` block

```swift
// Before (WRONG)
if showAICoach {
    ActivityAICoachSheet(...)
}
.onAppear { ... }  // ‚ùå Applied to 'if' result, not a View
```

**Solution**: Changed to use `.sheet()` modifier instead of conditional rendering

```swift
// After (CORRECT)
.sheet(isPresented: $showAICoach) {
    ActivityAICoachSheet(...)
}
.onAppear { ... }  // ‚úÖ Applied to NavigationStack
```

**Why it works**: 
- `.sheet()` properly returns a View that can have modifiers
- `.onAppear` is now applied to the NavigationStack, not a conditional
- Follows SwiftUI best practices for modal presentation

### 3. nonisolated(unsafe) Warning ‚úÖ

**Error**: `'nonisolated(unsafe)' has no effect on property 'statusCheckTimer', consider using 'nonisolated'`

**Location**: `ViewModels/RecipeExtractionViewModel.swift:18`

**Problem**: Using `nonisolated(unsafe)` on a stored property when not needed

```swift
// Before (unnecessary)
nonisolated(unsafe) private var statusCheckTimer: Timer?
```

**Solution**: Removed the annotation entirely

```swift
// After (correct)
private var statusCheckTimer: Timer?
```

**Why it works**: 
- The entire class is marked `@MainActor`
- Timer is already safe within MainActor context
- Timer always runs on main thread
- No need to bypass isolation checks

## üéØ KEY LEARNINGS

### 1. SwiftUI Modal Presentation
**Wrong**:
```swift
if showModal {
    ModalView()
}
.modifier { ... }  // ‚ùå Doesn't work
```

**Correct**:
```swift
.sheet(isPresented: $showModal) {
    ModalView()
}
.modifier { ... }  // ‚úÖ Works!
```

### 2. Type Ambiguity Resolution
- Sometimes simpler is better
- Direct type references can work better than aliases with macro systems
- If ambiguity persists, check for duplicate definitions

### 3. Swift Concurrency Annotations
- `@MainActor` on class = all members isolated
- `nonisolated(unsafe)` only for specific unsafe cases
- Timer within `@MainActor` context is already safe
- Use minimal annotations - let Swift infer safety
